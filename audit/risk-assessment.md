## 3. 버스 팩터 분석

현재 **MJ 1명**이 전체 코드베이스 소유:
- 147 커밋 전부 MJ (or MJ가 지시한 서브에이전트)
- 아키텍처 결정, 구현, 테스트 모두 1인
- ONE이 코드 리뷰하지만 깊은 코드 수준은 아닌 것으로 보임

**위험:** MJ에 장애 발생 시 프로젝트 진행 불가

**완화 방안:**
1. ✅ 이미 양호: 문서화 수준 높음 (MASTER-PLAN, PROGRESS, REQUIREMENTS)
2. ❌ 필요: 코드 레벨 문서 (TSDoc, 아키텍처 다이어그램)
3. ❌ 필요: 개발 환경 설정 가이드 (새 개발자 온보딩)
4. ❌ 필요: 의사결정 기록 (ADR — Architecture Decision Records)
